# UML Documentation

## Overview

This directory contains the Class Diagram of the project.

The class diagram represents the static structure of the system by defining classes, their attributes, methods, and relationships.

## Purpose

The class diagram is used to:

* Model the object-oriented structure of the system.
* Define relationships between classes.
* Support software design and development.
* Provide a reference for implementation.

## Contents

* Class Diagram

## Usage

Review the class diagram to understand the system's entities, responsibilities, and interactions before implementation.
